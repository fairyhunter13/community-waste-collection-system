# Phase 4 — Quality Assurance & Testing

## Purpose

Define the complete testing strategy across all three levels: unit, integration, and E2E. Tests are not an afterthought — they are written **alongside or immediately after** each sub-phase of Phase 3. This document also covers static analysis, benchmarks, coverage targets, and test tooling.

---

## 1. Testing Stack

| Tool | Purpose |
|---|---|
| `go test` | Native test runner; all tests use this |
| `testify/suite` | Test suite grouping; `SetupSuite`, `TearDownTest` lifecycle |
| `testify/assert` | Non-fatal assertions (test continues on failure) |
| `testify/require` | Fatal assertions (test stops on failure) |
| `testcontainers-go` | Spin up real PostgreSQL 17 containers for integration tests |
| `mockery` | Generate mocks from domain interfaces (unit tests) |
| Go build tags | Separate unit / integration / e2e test execution |

## 4. Unit Tests

**Scope:** Service layer business rules; handler HTTP contract.
**Dependencies:** All external dependencies mocked via interfaces.
**Build tag:** None (run by default).

### Mock Generation

```bash
go tool mockery --all --dir internal/domain --output internal/mocks --with-expecter
```

This generates mocks with `.On("Method", ...).Return(...)` API from all interfaces in `internal/domain/`.

Alternatively, write minimal hand-rolled mocks:
```go
type mockPickupRepo struct {
    mock.Mock
}

func (m *mockPickupRepo) HasPendingPaymentForHousehold(ctx context.Context, id uuid.UUID) (bool, error) {
    args := m.Called(ctx, id)
    return args.Bool(0), args.Error(1)
}
```

### Test Suite Setup (testify/suite)

```go
// internal/service/pickup_test.go
package service_test

import (
    "testing"
    "github.com/stretchr/testify/suite"
    "github.com/fairyhunter13/community-waste-collection-system/internal/mocks"
    "github.com/fairyhunter13/community-waste-collection-system/internal/service"
)

type PickupServiceUnitSuite struct {
    suite.Suite
    pickupRepo  *mocks.PickupRepository
    paymentRepo *mocks.PaymentRepository
    svc         domain.PickupService
}

func (s *PickupServiceUnitSuite) SetupTest() {
    s.pickupRepo  = mocks.NewPickupRepository(s.T())
    s.paymentRepo = mocks.NewPaymentRepository(s.T())
    s.svc = service.NewPickupService(s.pickupRepo, s.paymentRepo, nil)
}

func TestPickupServiceUnit(t *testing.T) {
    suite.Run(t, new(PickupServiceUnitSuite))
}
```

### Table-Driven Test Pattern

```go
func (s *PickupServiceUnitSuite) TestCreate() {
    tests := []struct {
        name       string
        setupMocks func()
        req        domain.CreatePickupRequest
        wantErr    error
        wantPickup bool
    }{
        {
            name: "BR-01: blocked by pending payment",
            setupMocks: func() {
                s.paymentRepo.On("HasPendingPaymentForHousehold", mock.Anything, mock.AnythingOfType("uuid.UUID")).
                    Return(true, nil)
            },
            req:     domain.CreatePickupRequest{HouseholdID: uuid.New(), Type: domain.WasteTypeOrganic},
            wantErr: domain.ErrConflict,
        },
        {
            name: "success: organic pickup created",
            setupMocks: func() {
                s.paymentRepo.On("HasPendingPaymentForHousehold", mock.Anything, mock.Anything).Return(false, nil)
                s.pickupRepo.On("Create", mock.Anything, mock.AnythingOfType("*domain.WastePickup")).Return(nil)
            },
            req:        domain.CreatePickupRequest{HouseholdID: uuid.New(), Type: domain.WasteTypeOrganic},
            wantPickup: true,
        },
    }

    for _, tt := range tests {
        s.Run(tt.name, func() {
            tt.setupMocks()
            pickup, err := s.svc.Create(s.T().Context(), tt.req)  // T.Context() — Go 1.24
            if tt.wantErr != nil {
                s.Require().Error(err)
                s.Require().ErrorIs(err, tt.wantErr)
            } else {
                s.Require().NoError(err)
                s.Require().NotNil(pickup)
            }
        })
    }
}
```

### Service Layer — What to Test

| Test | Business Rule | Expected Error |
|---|---|---|
| Create blocked by pending payment | BR-01 | `domain.ErrConflict` |
| Create succeeds (no pending payment) | — | nil |
| Schedule rejected: status not pending | BR-02 | `domain.ErrConflict` |
| Schedule rejected: electronic + no safety_check | BR-03 | `domain.ErrBusinessRule` |
| Schedule succeeds: pending + safety_check=true (electronic) | BR-02+03 | nil |
| Complete rejected: status not scheduled | — | `domain.ErrConflict` |
| Complete: auto-payment amount organic/plastic/paper = 50000 | BR-05 | nil |
| Complete: auto-payment amount electronic = 100000 | BR-05 | nil |
| Cancel rejected: status is completed | — | `domain.ErrConflict` |
| Cancel rejected: status is canceled | — | `domain.ErrConflict` |
| Payment confirm: missing file | BR-06 | `domain.ErrValidation` |
| Payment confirm: already paid | — | `domain.ErrConflict` |

### Handler Layer — What to Test

Use `httptest.NewRecorder()` and `echo.New()`:

```go
func (s *PickupHandlerUnitSuite) TestCreatePickup_RateLimitedResponse() {
    req := httptest.NewRequest(http.MethodPost, "/api/pickups", strings.NewReader(`{...}`))
    req.Header.Set("Content-Type", "application/json")
    rec := httptest.NewRecorder()
    c := s.echo.NewContext(req, rec)

    err := s.handler.CreatePickup(c)
    s.Require().NoError(err)
    s.Require().Equal(http.StatusCreated, rec.Code)

    var resp map[string]any
    json.Unmarshal(rec.Body.Bytes(), &resp)
    s.Require().True(resp["success"].(bool))
}
```

| Scenario | Expected HTTP Status |
|---|---|
| Valid body → service success | 201 (create) / 200 (update) |
| Invalid JSON body | 400 |
| Missing required field | 400 |
| Service returns ErrNotFound | 404 |
| Service returns ErrConflict | 409 |
| Service returns ErrBusinessRule | 422 |
| Service returns ErrInternalFailure | 500 |

## 5. Integration Tests

**Scope:** Repository layer against real PostgreSQL 17.
**Infrastructure:** testcontainers-go — one container per suite, reused across tests.
**Build tag:** `//go:build integration`

### Container Setup (shared across all repository suites)

```go
//go:build integration

package repository_test

import (
    "context"
    "testing"
    "github.com/testcontainers/testcontainers-go/modules/postgres"
    "github.com/stretchr/testify/suite"
)

type RepositorySuite struct {
    suite.Suite
    db        *sqlx.DB
    container *postgres.PostgresContainer
}

func (s *RepositorySuite) SetupSuite() {
    ctx := context.Background()
    container, err := postgres.RunContainer(ctx,
        testcontainers.WithImage("postgres:17-alpine"),
        postgres.WithDatabase("test_waste"),
        postgres.WithUsername("test"),
        postgres.WithPassword("test"),
        testcontainers.WithWaitStrategy(
            wait.ForLog("database system is ready to accept connections").WithOccurrence(2),
        ),
    )
    s.Require().NoError(err)
    s.container = container

    connStr, _ := container.ConnectionString(ctx, "sslmode=disable")
    s.db = repository.MustConnect(connStr)

    // Apply migrations
    m, _ := migrate.New("file://../../migrations", connStr)
    s.Require().NoError(m.Up())
}

func (s *RepositorySuite) TearDownTest() {
    // Truncate all tables after each test (preserve schema)
    _, err := s.db.ExecContext(context.Background(),
        `TRUNCATE households, waste_pickups, payments RESTART IDENTITY CASCADE`,
    )
    s.Require().NoError(err)
}

func (s *RepositorySuite) TearDownSuite() {
    s.db.Close()
    s.container.Terminate(context.Background())
}
```

### Integration Test Context (T.Context — Go 1.24)

```go
func (s *RepositorySuite) TestHouseholdRepository_FindByID_NotFound() {
    ctx := s.T().Context()  // context cancelled when test finishes
    _, err := s.repo.FindByID(ctx, uuid.New())
    s.Require().ErrorIs(err, domain.ErrNotFound)
}
```

### Repository Layer — What to Test

**Household repository:**
- `Create` → row exists in DB
- `FindByID` (found) → returns correct fields
- `FindByID` (not found) → `domain.ErrNotFound`
- `List` (empty) → empty slice, total=0
- `List` (paginated) → correct slice length and total
- `Delete` → row no longer exists

**Pickup repository:**
- `Create` → row exists with `pending` status
- `HasPendingPaymentForHousehold` with pending payment → true
- `HasPendingPaymentForHousehold` without → false
- `FindExpiredOrganic` → returns only organic, pending, created > cutoff
- `BulkCancel` → all specified IDs have `canceled` status
- `List` filtered by `status` and `household_id` → correct results
- `Schedule` → status = `scheduled`, pickup_date set

**Payment repository:**
- `CreateWithTx` within transaction → payment created atomically with pickup status change
- `Confirm` → status = `paid`, proof_file_url set, payment_date set
- `List` filtered by status, household, date range → correct results
- `WasteSummary` → correct aggregated counts per type+status
- `PaymentSummary` → correct counts + revenue total
- `HouseholdHistory` → returns household + pickups + payments

**Service layer integration (with real DB):**
- Complete pickup → verify pickup status + payment created in single TX; if TX fails, both roll back

## 6. E2E Tests

**Scope:** Full HTTP request lifecycle — real Echo server, real PostgreSQL, real MinIO.
**Infrastructure:** Local docker-compose stack (started before test run) or testcontainers orchestration.
**Build tag:** `//go:build e2e`

### E2E Suite Setup

```go
//go:build e2e

package e2e_test

type E2ESuite struct {
    suite.Suite
    baseURL string
    client  *http.Client
    db      *sqlx.DB
}

func (s *E2ESuite) SetupSuite() {
    s.baseURL = "http://localhost:8080"
    s.client  = &http.Client{Timeout: 10 * time.Second}
    s.db      = repository.MustConnect(os.Getenv("DATABASE_URL"))
}

func (s *E2ESuite) TearDownTest() {
    _, _ = s.db.Exec(`TRUNCATE households, waste_pickups, payments RESTART IDENTITY CASCADE`)
}
```

### E2E Scenarios

#### Full Pickup Lifecycle
```
POST /api/households → 201 (save household_id)
POST /api/pickups (household_id, type=plastic) → 201 (save pickup_id)
PUT /api/pickups/{pickup_id}/schedule → 200 (pickup_date required)
PUT /api/pickups/{pickup_id}/complete → 200
GET /api/payments → 200 (payment auto-created; save payment_id)
PUT /api/payments/{payment_id}/confirm (multipart file) → 200 (proof_file_url populated)
```

#### BR-01 E2E
```
POST /api/households → 201
POST /api/pickups (organic) → 201
PUT /api/pickups/:id/complete → 200 (payment pending)
POST /api/pickups (plastic, same household) → 409
```

#### BR-03 E2E
```
POST /api/pickups (electronic, safety_check=false) → 201
PUT /api/pickups/:id/schedule → 422
```

#### Rate Limit E2E
```
POST /api/pickups × 15 rapid requests → some 429 responses observed
```

#### Organic Worker E2E
```
INSERT pickup with created_at = NOW() - INTERVAL '4 days' (direct DB)
Wait for worker tick (or reduce interval in test env to 1s)
GET /api/pickups/:id → status = canceled
```

#### Reporting E2E
```
Create mixed household/pickup/payment data
GET /api/reports/waste-summary → verify counts match created data
GET /api/reports/payment-summary → verify revenue = expected
GET /api/reports/households/:id/history → verify all records present
```

#### Pagination E2E
```
Create 25 households
GET /api/households?per_page=20 → 20 results, total=25, total_pages=2
GET /api/households?page=2&per_page=20 → 5 results
```

## 7. Performance Tests

### 7a. Full-Stack HTTP Performance Tests (`test/perf/`)

**Scope:** End-to-end HTTP latency against the full running stack (app + postgres + minio + otel-collector).
**Build tag:** `//go:build perf`
**Requires:** `BASE_URL` env var pointing at the running app (e.g. `http://localhost:8080`).
**Run in CI:** `perf` job in `ci.yml` — spins up the full docker-compose stack identically to the `e2e` job.

```go
//go:build perf

package perf_test

import (
    "fmt"
    "net/http"
    "os"
    "strings"
    "testing"
)

func benchBaseURL(b *testing.B) string {
    url := os.Getenv("BASE_URL")
    if url == "" {
        b.Skip("BASE_URL not set — skipping performance tests")
    }
    return url
}

func BenchmarkListHouseholds(b *testing.B) { ... }   // GET /api/households
func BenchmarkCreateHousehold(b *testing.B) { ... }  // POST /api/households
func BenchmarkListPickups(b *testing.B) { ... }       // GET /api/pickups
func BenchmarkWasteSummary(b *testing.B) { ... }      // GET /api/reports/waste-summary
func BenchmarkPaymentSummary(b *testing.B) { ... }    // GET /api/reports/payment-summary
```

**Why HTTP-level instead of DB-level in CI:**
- Measures realistic user-facing latency (full round-trip: routing, middleware, DB, serialization)
- Catches regressions from any layer — a slow SQL query or expensive middleware both surface
- Results are actionable SLA numbers ("GET /api/reports/waste-summary: 3.2ms/op")

**Run locally:**
```bash
make perf   # requires full docker-compose stack running
```

## 10. Test Quality Rules

- **No `time.Sleep` in tests** — use testcontainers wait strategies or `testing/synctest` (Go 1.25)
- **No shared mutable state** between tests — each test is fully isolated via `TearDownTest` truncation
- **No hardcoded IDs** — use `uuid.New()` per test; never reuse IDs across tests
- **Descriptive test names** — `TestPickupService_Create_BlockedByPendingPayment` not `Test1`
- **Assert the right thing** — test the observable behavior, not the implementation
- **Don't test private functions** — test via the public interface; private functions are tested through their callers
- **Parallel where safe** — `t.Parallel()` is safe in unit tests; use carefully in integration tests (isolated DB state required)
