# Phase 4 â€” Quality Assurance & Testing

## Purpose

Define the complete testing strategy across all three levels: unit, integration, and E2E. Tests are not an afterthought â€” they are written **alongside or immediately after** each sub-phase of Phase 3. This document also covers static analysis, benchmarks, coverage targets, and test tooling.

---

## 1. Testing Stack

| Tool | Purpose |
|---|---|
| `go test` | Native test runner; all tests use this |
| `testify/suite` | Test suite grouping; `SetupSuite`, `TearDownTest` lifecycle |
| `testify/assert` | Non-fatal assertions (test continues on failure) |
| `testify/require` | Fatal assertions (test stops on failure) |
| `testcontainers-go` | Spin up real PostgreSQL 17 containers for integration tests |
| `mockery` | Generate mocks from domain interfaces (unit tests) |
| Go build tags | Separate unit / integration / e2e test execution |

## 4. Unit Tests

**Scope:** Service layer business rules; handler HTTP contract.
**Dependencies:** All external dependencies mocked via interfaces.
**Build tag:** None (run by default).

### Mock Generation

```bash
go tool mockery --all --dir internal/domain --output internal/mocks --with-expecter
```

This generates mocks with `.On("Method", ...).Return(...)` API from all interfaces in `internal/domain/`.

Alternatively, write minimal hand-rolled mocks:
```go
type mockPickupRepo struct {
    mock.Mock
}

func (m *mockPickupRepo) HasPendingPaymentForHousehold(ctx context.Context, id uuid.UUID) (bool, error) {
    args := m.Called(ctx, id)
    return args.Bool(0), args.Error(1)
}
```

### Test Suite Setup (testify/suite)

```go
// internal/service/pickup_test.go
package service_test

import (
    "testing"
    "github.com/stretchr/testify/suite"
    "github.com/fairyhunter13/community-waste-collection-system/internal/mocks"
    "github.com/fairyhunter13/community-waste-collection-system/internal/service"
)

type PickupServiceUnitSuite struct {
    suite.Suite
    pickupRepo  *mocks.PickupRepository
    paymentRepo *mocks.PaymentRepository
    svc         domain.PickupService
}

func (s *PickupServiceUnitSuite) SetupTest() {
    s.pickupRepo  = mocks.NewPickupRepository(s.T())
    s.paymentRepo = mocks.NewPaymentRepository(s.T())
    s.svc = service.NewPickupService(s.pickupRepo, s.paymentRepo, nil)
}

func TestPickupServiceUnit(t *testing.T) {
    suite.Run(t, new(PickupServiceUnitSuite))
}
```

### Table-Driven Test Pattern

```go
func (s *PickupServiceUnitSuite) TestCreate() {
    tests := []struct {
        name       string
        setupMocks func()
        req        domain.CreatePickupRequest
        wantErr    error
        wantPickup bool
    }{
        {
            name: "BR-01: blocked by pending payment",
            setupMocks: func() {
                s.paymentRepo.On("HasPendingPaymentForHousehold", mock.Anything, mock.AnythingOfType("uuid.UUID")).
                    Return(true, nil)
            },
            req:     domain.CreatePickupRequest{HouseholdID: uuid.New(), Type: domain.WasteTypeOrganic},
            wantErr: domain.ErrConflict,
        },
        {
            name: "success: organic pickup created",
            setupMocks: func() {
                s.paymentRepo.On("HasPendingPaymentForHousehold", mock.Anything, mock.Anything).Return(false, nil)
                s.pickupRepo.On("Create", mock.Anything, mock.AnythingOfType("*domain.WastePickup")).Return(nil)
            },
            req:        domain.CreatePickupRequest{HouseholdID: uuid.New(), Type: domain.WasteTypeOrganic},
            wantPickup: true,
        },
    }

    for _, tt := range tests {
        s.Run(tt.name, func() {
            tt.setupMocks()
            pickup, err := s.svc.Create(s.T().Context(), tt.req)  // T.Context() â€” Go 1.24
            if tt.wantErr != nil {
                s.Require().Error(err)
                s.Require().ErrorIs(err, tt.wantErr)
            } else {
                s.Require().NoError(err)
                s.Require().NotNil(pickup)
            }
        })
    }
}
```

### Service Layer â€” What to Test

| Test | Business Rule | Expected Error |
|---|---|---|
| Create blocked by pending payment | BR-01 | `domain.ErrConflict` |
| Create succeeds (no pending payment) | â€” | nil |
| Schedule rejected: status not pending | BR-02 | `domain.ErrConflict` |
| Schedule rejected: electronic + no safety_check | BR-03 | `domain.ErrBusinessRule` |
| Schedule succeeds: pending + safety_check=true (electronic) | BR-02+03 | nil |
| Complete rejected: status not scheduled | â€” | `domain.ErrConflict` |
| Complete: auto-payment amount organic/plastic/paper = 50000 | BR-05 | nil |
| Complete: auto-payment amount electronic = 100000 | BR-05 | nil |
| Cancel rejected: status is completed | â€” | `domain.ErrConflict` |
| Cancel rejected: status is canceled | â€” | `domain.ErrConflict` |
| Payment confirm: missing file | BR-06 | `domain.ErrValidation` |
| Payment confirm: already paid | â€” | `domain.ErrConflict` |

### Handler Layer â€” What to Test

Use `httptest.NewRecorder()` and `echo.New()`:

```go
func (s *PickupHandlerUnitSuite) TestCreatePickup_RateLimitedResponse() {
    req := httptest.NewRequest(http.MethodPost, "/api/pickups", strings.NewReader(`{...}`))
    req.Header.Set("Content-Type", "application/json")
    rec := httptest.NewRecorder()
    c := s.echo.NewContext(req, rec)

    err := s.handler.CreatePickup(c)
    s.Require().NoError(err)
    s.Require().Equal(http.StatusCreated, rec.Code)

    var resp map[string]any
    json.Unmarshal(rec.Body.Bytes(), &resp)
    s.Require().True(resp["success"].(bool))
}
```

| Scenario | Expected HTTP Status |
|---|---|
| Valid body â†’ service success | 201 (create) / 200 (update) |
| Invalid JSON body | 400 |
| Missing required field | 400 |
| Service returns ErrNotFound | 404 |
| Service returns ErrConflict | 409 |
| Service returns ErrBusinessRule | 422 |
| Service returns ErrInternalFailure | 500 |

## 5. Integration Tests

**Scope:** Repository layer against real PostgreSQL 17.
**Infrastructure:** testcontainers-go â€” one container per suite, reused across tests.
**Build tag:** `//go:build integration`

### Container Setup (shared across all repository suites)

```go
//go:build integration

package repository_test

import (
    "context"
    "testing"
    "github.com/testcontainers/testcontainers-go/modules/postgres"
    "github.com/stretchr/testify/suite"
)

type RepositorySuite struct {
    suite.Suite
    db        *sqlx.DB
    container *postgres.PostgresContainer
}

func (s *RepositorySuite) SetupSuite() {
    ctx := context.Background()
    container, err := postgres.RunContainer(ctx,
        testcontainers.WithImage("postgres:17-alpine"),
        postgres.WithDatabase("test_waste"),
        postgres.WithUsername("test"),
        postgres.WithPassword("test"),
        testcontainers.WithWaitStrategy(
            wait.ForLog("database system is ready to accept connections").WithOccurrence(2),
        ),
    )
    s.Require().NoError(err)
    s.container = container

    connStr, _ := container.ConnectionString(ctx, "sslmode=disable")
    s.db = repository.MustConnect(connStr)

    // Apply migrations
    m, _ := migrate.New("file://../../migrations", connStr)
    s.Require().NoError(m.Up())
}

func (s *RepositorySuite) TearDownTest() {
    // Truncate all tables after each test (preserve schema)
    _, err := s.db.ExecContext(context.Background(),
        `TRUNCATE households, waste_pickups, payments RESTART IDENTITY CASCADE`,
    )
    s.Require().NoError(err)
}

func (s *RepositorySuite) TearDownSuite() {
    s.db.Close()
    s.container.Terminate(context.Background())
}
```

### Integration Test Context (T.Context â€” Go 1.24)

```go
func (s *RepositorySuite) TestHouseholdRepository_FindByID_NotFound() {
    ctx := s.T().Context()  // context cancelled when test finishes
    _, err := s.repo.FindByID(ctx, uuid.New())
    s.Require().ErrorIs(err, domain.ErrNotFound)
}
```

### Repository Layer â€” What to Test

**Household repository:**
- `Create` â†’ row exists in DB
- `FindByID` (found) â†’ returns correct fields
- `FindByID` (not found) â†’ `domain.ErrNotFound`
- `List` (empty) â†’ empty slice, total=0
- `List` (paginated) â†’ correct slice length and total
- `Delete` â†’ row no longer exists

**Pickup repository:**
- `Create` â†’ row exists with `pending` status
- `HasPendingPaymentForHousehold` with pending payment â†’ true
- `HasPendingPaymentForHousehold` without â†’ false
- `FindExpiredOrganic` â†’ returns only organic, pending, created > cutoff
- `BulkCancel` â†’ all specified IDs have `canceled` status
- `List` filtered by `status` and `household_id` â†’ correct results
- `Schedule` â†’ status = `scheduled`, pickup_date set

**Payment repository:**
- `CreateWithTx` within transaction â†’ payment created atomically with pickup status change
- `Confirm` â†’ status = `paid`, proof_file_url set, payment_date set
- `List` filtered by status, household, date range â†’ correct results
- `WasteSummary` â†’ correct aggregated counts per type+status
- `PaymentSummary` â†’ correct counts + revenue total
- `HouseholdHistory` â†’ returns household + pickups + payments

**Service layer integration (with real DB):**
- Complete pickup â†’ verify pickup status + payment created in single TX; if TX fails, both roll back

## 10. Test Quality Rules

- **No `time.Sleep` in tests** â€” use testcontainers wait strategies or `testing/synctest` (Go 1.25)
- **No shared mutable state** between tests â€” each test is fully isolated via `TearDownTest` truncation
- **No hardcoded IDs** â€” use `uuid.New()` per test; never reuse IDs across tests
- **Descriptive test names** â€” `TestPickupService_Create_BlockedByPendingPayment` not `Test1`
- **Assert the right thing** â€” test the observable behavior, not the implementation
- **Don't test private functions** â€” test via the public interface; private functions are tested through their callers
- **Parallel where safe** â€” `t.Parallel()` is safe in unit tests; use carefully in integration tests (isolated DB state required)
